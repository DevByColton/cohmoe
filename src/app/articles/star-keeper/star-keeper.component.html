// MY GOAL FOR STAR KEEPER - MY OWN RPG

This version of Star Keeper was built using Monogame, written in C#. I set out to learn what goes into making the different systems of a game, not necessarily the graphics rendering. So, Monogame was a good choice for me because it handles some lower level graphics rendering. That meant I could focus on implementing my systems. This was my first project where I wanted to make systems that could actually support a game, rather than just throwing something together that works. This is a 2D project.

// SYSTEMS IMPLEMENTED

  • Input handling system, including mouse and keyboard, controllers, and different controller types (PS, Xbox, Nintendo, and generic). The input type can change in realtime, this can been seen in the Star Map when the glyphs change based on the input type
  • Generic control scheme, handling an array of control schemes that can change based on the active scene
  • Generic scenes and scene handling system
  • A camera system with resolution rendering and zoom
  • Circle and polygon collision detection
  • Entity world placement from json files

// GAMEPLAY VIDEO

The recording is all of the systems mentioned above working together. You start on your ship which has a planet map. On the planet map you can choose to travel between 4 planets. Each planet area acts as a parent scene, so it load and unloads assets and data as needed to ensure that nothing is loaded unnecessarily. There is no sound to the recording.

Sidenote, it probably goes without saying that the assets are placeholders. Besides the planet map, I'm actually pretty happy with that, considering my art skills.
