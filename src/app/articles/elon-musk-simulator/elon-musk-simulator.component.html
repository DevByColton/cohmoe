// ELON MUSK SIMULATOR

This is one of the first actual games I ever made. A couple years ago we were with some friends. I was talking about how I wanted to start making games. Somehow, the conversation lead to someone saying "It'd be funny if there was a game about Elon Musk shooting blunts out of his mouth at Tesla's and Space X vehicles." Naturally, this was the first game I decided to make. The only thing I had to decide on then was how the game would play. I decided that a Geometry Wars style knock off would be appropriate.

This was the first time I ever set out to make a game, so I had no expectation on how the code should be structured or perform. I just wanted it to work. This was also built using Monogame, written in C#. It is pretty straight forward, point the mouse, and click to shoot. There are some astroids and Space X vehicles. I never got around to the Tesla part.

The main learning of this project was mostly how the monogame framework worked, and how to use the content pipeline and sprite batching. This was also where I learned the basics of collision detection, making a camera that follows the player with world bounds, and the difference in projecting content to the world vs the screen.

In terms of gameplay, I was proud of myself for making some power ups that can stack. There are Speed, Big Blunts, and Blunts All Directions power ups. If the player already had the power up that was trying to be picked up, it would not activate it and wait for that type of power up to expire.
